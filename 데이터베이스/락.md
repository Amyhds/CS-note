- 데이터베이스에서 동시 접근으로 인한 데이터 충돌을 방지하는 기술
- 여러 사용자가 동시에 데이터베이스에 접근하여 데이터를 변경하거나 읽을 때, 락을 통해 데이터의 일관성을 유지하고 오류를 방지
- 성능이 중요하다면 락 사용을 최소화해야 함
#### 락의 작동 방식
1. **락 설정**: 트랜잭션이 시작될 때 필요한 데이터에 락을 설정합니다.
2. **데이터 접근**: 락이 설정된 데이터는 다른 트랜잭션이 접근하거나 변경할 수 없습니다.
3. **락 해제**: 트랜잭션이 종료될 때 락을 해제합니다.
